# Admin Portal ‚Äî Complete Reference

The admin portal (`http://localhost:8080`) is a full-featured web application for
bank IT administrators and branch managers to oversee the entire SAHAYAK kiosk network.

---

## Access

| Field    | Value                    |
|----------|--------------------------|
| URL      | http://localhost:8080    |
| Email    | admin@sahayak.com        |
| Password | admin123                 |
| Token    | 24-hour JWT (auto-stored)|

## Pages & Features

### 1. Dashboard

```
GET /api/admin/reports/kpi
GET /api/admin/reports/usage
GET /api/admin/reports/errors
GET /api/admin/kiosks
```

Shows live KPI cards:
- **Total Kiosks** ‚Äî registered kiosk count
- **Active Kiosks** ‚Äî heartbeat within last 2 minutes
- **Forms Today** ‚Äî submissions since midnight
- **Error Rate** ‚Äî percentage of failed submissions

Charts:
- Bar chart: forms processed by service category
- Line chart: daily error trend (last 7/30/90 days)
- Table: kiosk status overview

---

### 2. Kiosks

```
GET  /api/admin/kiosks            list all registered kiosks
POST /api/admin/kiosks            register new kiosk device
GET  /api/admin/kiosks/{id}       get single kiosk details
PUT  /api/admin/kiosks/{id}       update kiosk (branch, region)
DELETE /api/admin/kiosks/{id}     deregister kiosk
```

Status logic:
- **Online** ‚Äî `last_heartbeat` within past 2 minutes
- **Offline** ‚Äî `last_heartbeat` older than 2 minutes or null

Fields per kiosk:
- `device_id` ‚Äî unique identifier (generated by kiosk, stored in localStorage)
- `branch_name`, `branch_code` ‚Äî where it is physically located
- `region` ‚Äî for multi-region filtering
- `installed_version` ‚Äî reported by heartbeat
- `forms_today` ‚Äî incremented per submission
- `ip_address` ‚Äî last reported IP

---

### 3. Staff

```
GET    /api/admin/staff         list all staff users (returns plain array)
POST   /api/admin/staff         create new staff user + hashed PIN
PUT    /api/admin/staff/{id}    update staff name or role
DELETE /api/admin/staff/{id}    remove staff user
POST   /api/admin/staff/{id}/reset-pin   issue new 4-digit PIN
```

Each staff record:
- `name` ‚Äî staff member display name
- `role` ‚Äî "staff" | "manager" | "supervisor"
- `pin_hash` ‚Äî bcrypt hash of 4-digit PIN
- `created_at`

The kiosk uses staff PIN to authenticate form submissions via `POST /api/auth/staff-login`.

### 5. OTA Updates

```
GET    /api/admin/updates          list all OTA packages
POST   /api/admin/updates          create new update
PUT    /api/admin/updates/{id}     edit update details
DELETE /api/admin/updates/{id}     delete update
POST   /api/admin/updates/{id}/push   push to target device IDs
```

OTA Update fields:
- `version` ‚Äî e.g. `"1.1.0"`
- `update_name` ‚Äî human-readable name
- `notes` ‚Äî changelog / release notes
- `target` ‚Äî JSON array of device IDs or `["all"]`
- `status` ‚Äî Draft | Ready | Deployed

Kiosk side: On startup (mount), `AppStateContext.jsx` calls:
```
GET /api/kiosk/pending-update?device_id=<kiosk-id>
‚Üí { has_update: true, version: "1.1.0", update_name: "Security Patch" }
```
If `has_update`, the OTA Banner appears at the top of the kiosk screen.

### 6. Forms & Templates

```
GET    /api/admin/forms            list all form templates
POST   /api/admin/forms            create new template
GET    /api/admin/forms/{id}       get template with field definitions
PUT    /api/admin/forms/{id}       update template
DELETE /api/admin/forms/{id}       delete template
POST   /api/admin/forms/ocr        OCR scan a document image ‚Üí auto-fill fields
```

Each template has:
- `name` ‚Äî display name
- `category` ‚Äî internal key (must match `FORM_TEMPLATES` keys in `forms.py`)
- `version` ‚Äî integer, auto-incremented on update
- `status` ‚Äî Draft | Published
- `languages` ‚Äî supported languages
- `field_definitions` ‚Äî JSON array of field objects

Field object structure:
```json
{
  "id": "accountNumber",
  "label": "Account Number",
  "type": "tel",
  "required": true,
  "placeholder": "Enter 16-digit account number"
}
```

Supported field types: `text`, `tel`, `number`, `email`, `date`, `select`

The **Visual Field Builder** in the admin UI allows drag-and-add field creation
with a live preview of the form.

### 9. Settings

```
GET /api/admin/settings       returns all system settings as key:value
PUT /api/admin/settings       update one or more settings
```

Available settings:

| Key                    | Type    | Default | Description                        |
|------------------------|---------|---------|------------------------------------|
| `offlineMode`          | boolean | `true`  | Kiosk works without internet       |
| `hybridMode`           | boolean | `false` | Mix of online + offline operations |
| `voiceVerification`    | boolean | `true`  | Enable voice in human verification |
| `autoSync`             | boolean | `true`  | Auto-sync to backend periodically  |
| `syncIntervalMinutes`  | number  | `30`    | How often to sync (minutes)        |
| `dataRetentionDays`    | number  | `90`    | Days to retain submission records  |

## Admin Authentication Technical Details

```
POST /api/admin/auth/login
Body: { "email": "admin@sahayak.com", "password": "admin123" }
Response: {
  "access_token": "<JWT>",
  "token_type": "bearer",
  "admin": {
    "id": 1, "name": "Super Admin", "email": "...", "role": "Super Admin"
  }
}

Token payload:
{
  "sub": "1",
  "exp": <unix timestamp 24h>,
  "type": "admin"
}

All admin endpoints require:
  Authorization: Bearer <access_token>
```

Admin tokens use a SEPARATE secret (`ADMIN_JWT_SECRET`) from kiosk staff tokens
(`JWT_SECRET`) ‚Äî they cannot be cross-used.

## API Client (Sahayak_admin/src/utils/apiClient.js)

The admin frontend uses an Axios-based API client with:
1. **Base URL:** `/api` (relative ‚Äî proxied by Nginx to `backend:5000`)
2. **Auto-inject JWT:** reads from `localStorage.getItem('sahayak_admin_token')`
3. **401 handler:** on any 401 that is NOT the login endpoint itself, clears token
   and redirects to `/login`

```javascript
// Key fix: skip redirect when it IS the login call
if (error.response?.status === 401) {
    const path = error.config?.url || '';
    if (!path.includes('auth/login')) {
        clearToken();
        window.location.href = '/login';
    }
}
```

## Audit Logging

Every mutation (POST/PUT/DELETE) on admin endpoints creates an `audit_log` record:

```python
db.add(AuditLog(
    admin_user_id=current_admin.id,
    action="CREATE",
    resource="staff_user",
    resource_id=str(new_staff.id),
    detail={"name": body.name, "role": body.role},
    ip_address=request.client.host
))
```

Audit logs are viewable in the database:
```sql
SELECT * FROM audit_logs ORDER BY timestamp DESC LIMIT 50;
```
