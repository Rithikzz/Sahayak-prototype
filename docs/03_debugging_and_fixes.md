# Debugging & Error Resolution Guide

This document captures historical fixes and how to handle specialized Edge Cases within the SAHAYAK Python backend application.

## 1. 0-Byte Audio Stream Proxy Drop
### Symptoms:
The STT service was returning an empty `{"text": "", "language": "en"}` response instantly, forcing the frontend UI into an infinite loop waiting for text.
### The Fix implemented:
`SpooledTemporaryFile` objects generated by FastAPI's `UploadFile` must be fully materialized into memory chunks before being forwarded through an asynchronous downstream request (like `httpx`). In `voice.py`:
```python
@router.post("/transcribe")
async def transcribe_audio(audio: UploadFile = File(...)):
    # We must explicitly read the buffer
    audio_bytes = await audio.read() 
    # Use explicit bytes instead of implicitly proxying the file handle
    files = {'file': (audio.filename, audio_bytes, audio.content_type)}
    # ...
```

## 2. Passlib vs Bcrypt Library Issue
### Symptoms:
During `staff_login`, hashing passwords failed with `ValueError: password cannot be longer than 72 bytes`.
### The Fix implemented:
The standard `passlib` version is somewhat unsupported on modern Python 3.11 when enforcing Bcrypt rules. I stripped `passlib` from the `requirements.txt` entirely, instead opting for the pure `bcrypt` binary package native to Python architecture, resolving password length limits safely without rewriting the auth architecture.

## 3. UI "Loading Forms..." Infinite Screen
### Symptoms:
The React app froze immediately after staff login, preventing users from seeing the Kiosk Service Selection Screen.
### The Fix implemented:
To make the application dynamic, `mockData.js` was disconnected and the app was instructed to read the form fields schema dynamically from the Python JSON backend. However, a React rendering effect was originally missing. We added:
```javascript
  useEffect(() => {
    const fetchTemplates = async () => {
      // ... await fetch('/api/forms/templates')
    };
    fetchTemplates();
  }, []);
```
To the root level `AppStateContext.jsx` file to cleanly hook the API into the global UI state engine!

## 4. Voice "Auto-Advance" IVR
### Symptoms:
Users had to manually click "Next" aggressively on the touch screen even while speaking to the system. If they stopped speaking, it required a manual click to record again.
### The Fix implemented:
The audio pipeline (`InputController.jsx`) was transitioned to a true Continuous IVR flow.
1. The Kiosk speaks using the TTS proxy hook.
2. The UI reads an `audio.onended` internal state and **automatically** begins listening on the local device microphone for 5 seconds.
3. The UI automatically uploads the `.webm` blob.
4. If STT returns transcribed text, it saves to form state. If STT translates silence, it logs `[No Speech Detected]`.
5. It then **automatically injects a 1500ms delay** and then dynamically navigates the `currentFieldIndex` forward without any user touch input!
